## Overview

The `LDAP::Server::Schema::AttributeType` class represents an LDAP AttributeTypeDescription as defined in RFC2252 section 4.2. It encapsulates all properties of an attribute type including OID, names, syntax, matching rules, and constraints.

## Class Methods

### new

Creates a new AttributeType instance by parsing an RFC2252-compliant attribute type definition string.

<ParamField path="str" type="string" required>
  RFC2252-compliant AttributeTypeDescription string
</ParamField>

```ruby
attr = LDAP::Server::Schema::AttributeType.new(
  "( 2.5.4.3 NAME ( 'cn' 'commonName' ) DESC 'RFC2256: common name(s)' SUP name )"
)
```

**Raises:** `LDAP::ResultError::InvalidAttributeSyntax` if the definition string is malformed

## Instance Methods

### name

Returns the primary name of the attribute (the first name in the names array).

```ruby
attr = LDAP::Server::Schema::AttributeType.new(
  "( 2.5.4.3 NAME ( 'cn' 'commonName' ) DESC 'common name' SUP name )"
)
puts attr.name  # => "cn"
```

**Returns:** String - The primary attribute name, or nil if no names are defined

### to_s

Returns a string representation of the attribute, using the primary name if available, otherwise the OID.

```ruby
puts attr.to_s  # => "cn"
```

**Returns:** String - The primary name or OID

### to_def

Returns the complete RFC2252-compliant definition string for this attribute type. The result is cached until `changed` is called.

```ruby
attr_def = attr.to_def
# => "( 2.5.4.3 NAME ( 'cn' 'commonName' ) DESC 'RFC2256: common name(s)' SUP name )"
```

**Returns:** String - RFC2252-compliant AttributeTypeDescription

### changed

Clears the cached definition string, forcing it to be rebuilt on the next call to `to_def`.

```ruby
attr.changed
```

## Attribute Readers

All properties of the attribute type are accessible via reader methods:

### oid

The object identifier (OID) for this attribute type.

```ruby
puts attr.oid  # => "2.5.4.3"
```

**Returns:** String - The OID

### names

Array of all names and aliases for this attribute type.

```ruby
puts attr.names.inspect  # => ["cn", "commonName"]
```

**Returns:** Array of strings

### desc

The description text for this attribute type.

```ruby
puts attr.desc  # => "RFC2256: common name(s) for which the entity is known by"
```

**Returns:** String or nil

### obsolete

Indicates whether this attribute type is marked as obsolete.

```ruby
puts attr.obsolete  # => false
```

**Returns:** Boolean

### sup

The superior attribute type from which this attribute inherits properties. After `resolve_oids` is called on the schema, this becomes an AttributeType object rather than a string.

```ruby
puts attr.sup  # => "name" (before resolve_oids)
# or
puts attr.sup.name  # => "name" (after resolve_oids)
```

**Returns:** String, AttributeType instance, or nil

### equality

The equality matching rule OID or name. After `resolve_oids` is called on the schema, this becomes a MatchingRule object.

```ruby
puts attr.equality  # => "caseIgnoreMatch"
```

**Returns:** String, MatchingRule instance, or nil

### ordering

The ordering matching rule OID or name. After `resolve_oids` is called on the schema, this becomes a MatchingRule object.

```ruby
puts attr.ordering  # => "caseIgnoreOrderingMatch"
```

**Returns:** String, MatchingRule instance, or nil

### substr

The substring matching rule OID or name. After `resolve_oids` is called on the schema, this becomes a MatchingRule object.

```ruby
puts attr.substr  # => "caseIgnoreSubstringsMatch"
```

**Returns:** String, MatchingRule instance, or nil

### syntax

The LDAP syntax OID for values of this attribute. After `resolve_oids` is called on the schema, this becomes a Syntax object.

```ruby
puts attr.syntax  # => "1.3.6.1.4.1.1466.115.121.1.15"
```

**Returns:** String, Syntax instance, or nil

### maxlen

The maximum length constraint for attribute values.

```ruby
puts attr.maxlen  # => 32768
```

**Returns:** Integer or nil

### singlevalue

Indicates whether this attribute can have only a single value (true) or multiple values (false).

```ruby
puts attr.singlevalue  # => false
```

**Returns:** Boolean

### collective

Indicates whether this is a collective attribute.

```ruby
puts attr.collective  # => false
```

**Returns:** Boolean

### nousermod

Indicates whether user modification of this attribute is forbidden (NO-USER-MODIFICATION).

```ruby
puts attr.nousermod  # => false
```

**Returns:** Boolean

### usage

The attribute usage type as a symbol.

```ruby
puts attr.usage  # => :userApplications
# or :directoryOperation, :distributedOperation, :dSAOperation
```

**Returns:** Symbol or nil - One of `:userApplications`, `:directoryOperation`, `:distributedOperation`, `:dSAOperation`

## Example Usage

```ruby
require 'ldap/server/schema'

# Create an attribute type from a definition
attr_def = <<~DEF
  ( 2.5.4.3 
    NAME ( 'cn' 'commonName' )
    DESC 'RFC2256: common name(s) for which the entity is known by'
    SUP name
  )
DEF

attr = LDAP::Server::Schema::AttributeType.new(attr_def)

# Access properties
puts "Name: #{attr.name}"
puts "OID: #{attr.oid}"
puts "All names: #{attr.names.join(', ')}"
puts "Description: #{attr.desc}"
puts "Superior: #{attr.sup}"
puts "Single-valued: #{attr.singlevalue}"
puts "User-modifiable: #{!attr.nousermod}"

# Get the complete definition
puts attr.to_def
```

## Property Inheritance

When an attribute type has a superior (`SUP`), it inherits properties from that superior during the `resolve_oids` phase:

- equality
- ordering
- substr
- syntax
- maxlen
- singlevalue
- collective
- nousermod
- usage

Explicitly defined properties take precedence over inherited ones.

```ruby
schema = LDAP::Server::Schema.new
schema.add_attrtype("( 2.5.4.41 NAME 'name' EQUALITY caseIgnoreMatch SYNTAX 1.3.6.1.4.1.1466.115.121.1.15 )")
schema.add_attrtype("( 2.5.4.3 NAME 'cn' SUP name )")
schema.resolve_oids

cn_attr = schema.find_attrtype('cn')
puts cn_attr.equality  # Inherited from 'name': caseIgnoreMatch
```
